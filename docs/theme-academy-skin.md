# 从前书院皮肤说明

更新时间：2026-04-09

## 目标

通过 `?theme=academy` 为黄诗扶生日活动提供一套独立皮肤，不改后端，仅在前端切换视觉与交互状态与可见入口。

## 规则

- `?theme=academy` 优先级最高。
- URL 没有 `theme` 时，读取本地存储偏好。
- 仅接受 `default` 和 `academy` 两个值。
- academy 模式下，自动进入无感浏览状态：隐藏登录/注册/个人资料/管理入口，并通过路由守卫拦截写操作页面。

## 已完成范围（任务 1-5）

### 1) 主题基础设施与导航保持

- 启动时先应用 `data-theme`，减少闪烁。
- `src/lib/theme.ts` 统一处理解析、持久化、URL 主题参数合并。
- `ThemeContext` 提供 `theme` / `isAcademy` / `toggleTheme`。
- 站内跳转统一通过主题参数保留逻辑（academy 下持续保留 `theme=academy`）。

### 2) 可见切换与路由守卫行为

- 顶部导航提供可见的主题切换按钮（默认/书院互切）。
- academy 模式下隐藏默认主题写入入口与账号相关入口。
- 路由层对不符合 academy 浏览态的页面进行守卫/重定向，保留只读浏览路径。

### 3) 书院视觉区分（不影响默认主题）

- 首页、导航、公告栏增加 academy 专属视觉层（`[data-theme='academy']` + `academy-*` 类）。
- 书院视觉包含纸面/题签/边框风格增强，默认主题在触达区域保持原样。

### 4) 任务 5 补齐模块

- 公告栏：academy 下明确红色抬头（notice-board）处理。
- 首页：academy-only 彩蛋面板（可发现按钮触发）。
- 首页/Wiki/Forum：academy-only 讲师卡片与文案映射表（默认文案 vs 书院文案）。

## 明确非目标（本轮未做）

- 不修改后端/API/Prisma/server 逻辑。
- 不改默认主题信息架构与业务路径。
- 不新增依赖、不新增后端配置项。
- 不引入新的产品需求，仅完成既定书院皮肤范围。

## 验证方式

### 自动验证（必须）

```bash
npm run build
```

### 手工冒烟（由用户执行）

1. 在默认主题打开首页/Wiki/Forum，确认外观与入口行为不变。
2. 切换到 academy：
   - 顶部主题标识与书院风格生效；
   - 公告栏出现红色抬头样式；
   - 首页可展开彩蛋面板；
   - 首页/Wiki/Forum出现讲师卡片与映射表。
3. academy 模式下验证隐藏入口与路由守卫仍生效。

> 说明：本轮手工 UI 校验由用户负责，文档仅给出冒烟步骤与核对点。

## 风险与后续

- academy 样式依赖主题参数保留；若手动清理 URL 参数将回退默认主题。
- 后续若扩展 academy 页面，需保持同样约束：academy-only 样式与逻辑不外溢到默认主题。
