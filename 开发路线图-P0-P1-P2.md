# 诗扶小筑开发路线图（P0 / P1 / P2）

本文档基于当前仓库现状（React + Express + Prisma + MySQL）整理，目标是将现有能力从 MVP 原型推进到可运营、可增长、可扩展的产品形态。

---

## P0（必须优先，形成可运营闭环）

- 目标：打通“用户投稿 -> 审核 -> 发布 -> 互动”全链路。
- 时间建议：2~3 周（单人全栈）。

### 1) 审核发布流（Wiki / 帖子）

#### 数据库变更

- `WikiPage` 新增字段：
  - `status enum('draft','pending','published','rejected') default 'published'`
  - `reviewNote text?`
  - `reviewedBy string?`
  - `reviewedAt datetime?`
- `Post` 新增字段：
  - `status enum('draft','pending','published','rejected') default 'published'`
  - `reviewNote text?`
  - `reviewedBy string?`
  - `reviewedAt datetime?`
- 新增表 `ModerationLog`：
  - `id`
  - `targetType`（`wiki` / `post`）
  - `targetId`
  - `action`（`submit` / `approve` / `reject` / `rollback`）
  - `operatorUid`
  - `note`
  - `createdAt`

#### 接口清单

- `POST /api/wiki/:slug/submit`（用户提交审核）
- `POST /api/posts/:id/submit`
- `GET /api/admin/review-queue?type=wiki|posts&status=pending`
- `POST /api/admin/review/:type/:id/approve`
- `POST /api/admin/review/:type/:id/reject`

#### 前端任务

- Wiki/Forum 编辑页加入“保存草稿”和“提交审核”按钮。
- Admin 新增“审核队列”Tab。
- 非 `published` 内容对游客不可见。

#### 验收标准

- 普通用户不能直接发布。
- 管理员审核通过后内容可公开。
- 审核动作全量记录，可追溯。

---

### 2) 收藏与点赞（真实可用）

#### 数据库变更

- 新增表 `PostLike`：
  - `id`
  - `postId`
  - `userUid`
  - `createdAt`
  - 唯一索引：`postId + userUid`
- 新增表 `Favorite`：
  - `id`
  - `userUid`
  - `targetType`（`wiki` / `post` / `music`）
  - `targetId`
  - `createdAt`
  - 唯一索引：`userUid + targetType + targetId`
- `WikiPage` 新增：`favoritesCount int default 0`
- `Post.likesCount` 由点赞关系表驱动（保留计数缓存字段）。

#### 接口清单

- `POST /api/posts/:id/like`
- `DELETE /api/posts/:id/like`
- `POST /api/favorites`
- `DELETE /api/favorites/:type/:id`
- `GET /api/users/me/favorites?type=wiki|post|music`

#### 前端任务

- Forum 详情页点赞按钮接入真实接口。
- Wiki/帖子/音乐卡片加入收藏按钮。
- Profile 新增“我的收藏”页签。

#### 验收标准

- 同一用户不可重复点赞/收藏。
- 计数准确、可取消、刷新后状态一致。

---

### 3) 用户封禁与基础风控

#### 数据库变更

- `User` 新增字段：
  - `status enum('active','banned') default 'active'`
  - `banReason text?`
  - `bannedAt datetime?`

#### 接口清单

- `POST /api/admin/users/:uid/ban`
- `POST /api/admin/users/:uid/unban`

#### 前端任务

- Admin 用户管理增加封禁/解封按钮。
- 被封禁用户禁止发帖/评论/编辑。

#### 验收标准

- 封禁后写操作被拒绝（读权限按策略保留或限制）。
- 管理员可恢复并记录操作日志。

---

## P1（增长与体验，提升留存）

- 目标：强化发现效率、互动反馈和个人沉淀。
- 时间建议：2~4 周。

### 1) 推荐与热门排序

#### 数据库变更

- `Post` 新增 `hotScore float default 0`
- `WikiPage` 新增 `viewCount int default 0`
- 可选新增 `ContentStatDaily`（按天聚合统计）

#### 接口清单

- `GET /api/posts?sort=latest|hot|recommended`
- `GET /api/wiki/recommended?slug=...`
- `GET /api/home/feed`

#### 算法建议（先简版）

- `hotScore = likes*3 + comments*2 + views*0.2 + timeDecay`

#### 前端任务

- Forum 增加“最新 / 热门 / 推荐”切换。
- 首页改为聚合接口输出（动态、推荐 Wiki、热门音乐、新增内容）。

#### 验收标准

- “热门”和“最新”结果明显不同，且分数可解释。

---

### 2) 通知系统（站内）

#### 数据库变更

- 新增表 `Notification`：
  - `id`
  - `userUid`
  - `type`（reply/like/review_result 等）
  - `payload json`
  - `isRead boolean`
  - `createdAt`

#### 接口清单

- `GET /api/notifications`
- `POST /api/notifications/read-all`
- `POST /api/notifications/:id/read`

#### 触发事件

- 评论被回复。
- 帖子被点赞。
- 投稿审核结果变更。

#### 前端任务

- Navbar 增加通知红点和通知抽屉。

#### 验收标准

- 关键互动均能生成通知。
- 支持未读/已读状态管理。

---

### 3) 用户行为中心（Profile 完整化）

#### 数据库变更

- 新增表 `BrowsingHistory`：
  - `id`
  - `userUid`
  - `targetType`
  - `targetId`
  - `createdAt`

#### 接口清单

- `GET /api/users/:uid/posts`
- `GET /api/users/:uid/comments`
- `GET /api/users/me/history`

#### 前端任务

- Profile 落地“发帖记录 / 评论记录 / 浏览历史”。

#### 验收标准

- 个人中心不再是占位内容，数据可持续沉淀。

---

### 4) 搜索增强

#### 数据库变更（可选）

- 新增表 `SearchKeyword`：
  - `keyword`
  - `count`
  - `updatedAt`

#### 接口清单

- `GET /api/search/hot-keywords`
- `GET /api/search/suggest?q=...`

#### 前端任务

- 热搜词改为实时统计。
- 搜索输入框增加联想建议。

#### 验收标准

- 热搜不再硬编码，联想命中率可观。

---

## P2（差异化与平台扩展）

- 目标：建立平台壁垒与多端增长通道。
- 时间建议：3~6 周。

### 1) 微信登录 + 小程序精简端

#### 数据库变更

- `User` 新增：
  - `wechatOpenId unique?`
  - `wechatUnionId?`

#### 接口清单

- `POST /api/auth/wechat/login`（code 换 openid / unionid）
- 小程序专用接口：
  - `GET /api/mp/wiki`
  - `POST /api/mp/posts`
  - `POST /api/mp/comments`

#### 客户端任务

- 新建小程序工程，先实现：浏览 Wiki、发帖、评论。

#### 验收标准

- 微信登录可用，账号关系可绑定。
- 小程序完成最小闭环。

---

### 2) 作品数据库（歌曲 / 专辑独立建模）

#### 数据库变更

- 新增模型：
  - `Artist`
  - `Album`
  - `Track`
  - `TrackInAlbum`（多对多关联）

#### 接口清单

- `GET /api/albums`
- `GET /api/albums/:id`
- `GET /api/tracks`
- `POST /api/admin/tracks/import`

#### 前端任务

- 新增专辑页、曲目页和 Wiki 关联入口。

#### 验收标准

- “专辑系统”从 Wiki 分类升级为结构化数据库能力。

---

### 3) AI 生产力增强

#### 数据库变更

- 新增表 `AiJob`：
  - `id`
  - `type`
  - `status`
  - `input`
  - `output`
  - `createdBy`
  - `createdAt`

#### 接口清单

- `POST /api/ai/wiki/auto-intro`
- `POST /api/ai/wiki/auto-summary`
- `POST /api/ai/moderation/risk-check`

#### 前端任务

- 编辑器内接入 AI 草稿、摘要、风险提示。

#### 验收标准

- 明显缩短编辑时间，且 AI 产出可追踪、可审计。

---

## 推荐实施节奏（Sprint 参考）

- Sprint 1：P0 上半（审核流 + 后台审核队列）
- Sprint 2：P0 下半（点赞收藏 + 封禁 + Profile 收藏）
- Sprint 3：P1（热门推荐 + 通知 + 用户行为中心）
- Sprint 4：P2（微信/小程序 或 作品数据库，二选一先做）

---

## 里程碑验收总表

- M1（P0 完成）：可审核发布、可点赞收藏、可封禁治理。
- M2（P1 完成）：可推荐分发、可通知反馈、可行为沉淀。
- M3（P2 完成）：可多端增长、可结构化作品运营、可 AI 提效。
