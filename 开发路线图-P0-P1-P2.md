# 诗扶小筑开发路线图（P0 / P1 / P2）

本文档基于当前仓库现状（React + Express + Prisma + MySQL）整理，目标是将现有能力从 MVP 原型推进到可运营、可增长、可扩展的产品形态。

---

## P0（必须优先，形成可运营闭环）

- 目标：打通"用户投稿 -> 审核 -> 发布 -> 互动"全链路。
- 时间建议：2~3 周（单人全栈）。

### 1) 审核发布流（Wiki / 帖子）

#### 数据库变更

- `WikiPage` 新增字段：
  - `status enum('draft','pending','published','rejected') default 'published'`
  - `reviewNote text?`
  - `reviewedBy string?`
  - `reviewedAt datetime?`
- `Post` 新增字段：
  - `status enum('draft','pending','published','rejected') default 'published'`
  - `reviewNote text?`
  - `reviewedBy string?`
  - `reviewedAt datetime?`
- 新增表 `ModerationLog`：
  - `id`
  - `targetType`（`wiki` / `post`）
  - `targetId`
  - `action`（`submit` / `approve` / `reject` / `rollback`）
  - `operatorUid`
  - `note`
  - `createdAt`

#### 接口清单

- `POST /api/wiki/:slug/submit`（用户提交审核）
- `POST /api/posts/:id/submit`
- `GET /api/admin/review-queue?type=wiki|posts&status=pending`
- `POST /api/admin/review/:type/:id/approve`
- `POST /api/admin/review/:type/:id/reject`

#### 前端任务

- Wiki/Forum 编辑页加入"保存草稿"和"提交审核"按钮。
- Admin 新增"审核队列"Tab。
- 非 `published` 内容对游客不可见。

#### 验收标准

- 普通用户不能直接发布。
- 管理员审核通过后内容可公开。
- 审核动作全量记录，可追溯。

---

### 2) 收藏与点赞（真实可用）

#### 数据库变更

- 新增表 `PostLike`：
  - `id`
  - `postId`
  - `userUid`
  - `createdAt`
  - 唯一索引：`postId + userUid`
- 新增表 `Favorite`：
  - `id`
  - `userUid`
  - `targetType`（`wiki` / `post` / `music`）
  - `targetId`
  - `createdAt`
  - 唯一索引：`userUid + targetType + targetId`
- `WikiPage` 新增：`favoritesCount int default 0`
- `Post.likesCount` 由点赞关系表驱动（保留计数缓存字段）。

#### 接口清单

- `POST /api/posts/:id/like`
- `DELETE /api/posts/:id/like`
- `POST /api/favorites`
- `DELETE /api/favorites/:type/:id`
- `GET /api/users/me/favorites?type=wiki|post|music`

#### 前端任务

- Forum 详情页点赞按钮接入真实接口。
- Wiki/帖子/音乐卡片加入收藏按钮。
- Profile 新增"我的收藏"页签。

#### 验收标准

- 同一用户不可重复点赞/收藏。
- 计数准确、可取消、刷新后状态一致。

---

### 3) 用户封禁与基础风控

#### 数据库变更

- `User` 新增字段：
  - `status enum('active','banned') default 'active'`
  - `banReason text?`
  - `bannedAt datetime?`

#### 接口清单

- `POST /api/admin/users/:uid/ban`
- `POST /api/admin/users/:uid/unban`

#### 前端任务

- Admin 用户管理增加封禁/解封按钮。
- 被封禁用户禁止发帖/评论/编辑。

#### 验收标准

- 封禁后写操作被拒绝（读权限按策略保留或限制）。
- 管理员可恢复并记录操作日志。

## P1（增长与体验，提升留存）

- 目标：强化发现效率、互动反馈和个人沉淀。
- 时间建议：2~4 周。
- **完成状态**：✅ 已完成（2026-03-25）

### 1) 推荐与热门排序

#### 数据库变更

- `Post` 新增 `hotScore float default 0` ✅
- `WikiPage` 新增 `viewCount int default 0` ✅

#### 接口清单

- `GET /api/posts?sort=latest|hot|recommended` ✅ 已实现（server.ts，含热分数实时重算）
- `GET /api/wiki/recommended?slug=...` ✅ 已实现（基于分类标签打分的内容推荐）
- `GET /api/home/feed` ✅ 已实现（公告 + 热门帖子 + 最新帖子聚合）

#### 算法建议（先简版）

- `hotScore = likes*3 + comments*2 + views*0.2 + timeDecay` ✅ 已实现（`calculatePostHotScore()` 函数）

#### 前端任务

- Forum 增加"最新 / 热门 / 推荐"切换。 ✅ 已实现（Forum.tsx 增加 sort tabs）
- 首页改为聚合接口输出。 ✅ 已实现（Home.tsx 改为调用 /api/home/feed）

#### 验收标准

- "热门"和"最新"结果明显不同，且分数可解释。 ✅ 已实现

## P1 实施详情（2026-03-25）

### Backend 变更

**server.ts 新增接口：**
- `GET /api/notifications` — 通知列表，含分页和未读计数
- `POST /api/notifications/:id/read` — 标记单条通知已读
- `POST /api/notifications/read-all` — 全部标记已读
- `GET /api/users/:uid/posts` — 用户帖子列表
- `GET /api/users/:uid/comments` — 用户评论列表
- `GET /api/users/me/history` — 浏览历史（支持 type 筛选）
- `GET /api/home/feed` — 首页聚合（公告 + 热门 + 最新）
- `GET /api/search/hot-keywords` — 热搜词 Top 20
- `GET /api/search/suggest?q=` — 搜索联想（keyword/wiki/post 各 3 条）

**server.ts 新增辅助函数：**
- `toNotificationResponse()` — 通知响应格式化
- `createNotification()` — 创建通知（异步，异常不阻断主流程）
- `recordBrowsingHistory()` — 记录浏览历史（30min 去重）
- `increaseSearchKeywordCount()` — 搜索词计数（upsert）
- `refreshPostHotScore()` — 热分数刷新（使用 `$queryRaw`/`$executeRaw` 绕过 Prisma 类型限制）
- `normalizeKeyword()` — 搜索词归一化

**prisma/schema.prisma 变更：**
- 新增 `NotificationType` enum
- 新增 `BrowsingTargetType` enum
- 新增 `Notification` 模型
- 新增 `BrowsingHistory` 模型
- 新增 `SearchKeyword` 模型
- `Post` 新增 `hotScore Float` 和 `viewCount Int`
- `WikiPage` 新增 `viewCount Int`

### Frontend 变更

**Navbar.tsx：**
- 新增通知铃铛图标（Bell icon）
- 新增未读数红点badge（最多显示 99+）
- 新增通知下拉面板（最多显示 10 条）
- 支持点击单条标记已读、支持"全部已读"
- 自动轮询间隔 60 秒

**Forum.tsx：**
- 新增 sort 查询参数支持（latest/hot/recommended）
- Forum 列表页新增排序 Tab 切换（最新/热门/推荐）
- 帖子列表 API 调用传递 sort 参数

**Home.tsx：**
- 移除旧的 `/api/posts` 直接调用
- 改为调用 `/api/home/feed` 获取聚合数据
- 显示 announcement、hotPosts、recentPosts

**Profile.tsx：**
- 移除 Firebase compat 的 `updateDoc` 调用，改为 `apiPatch('/api/users/me', ...)`
- 新增"我的帖子"Tab（调用 `/api/users/:uid/posts`）
- 新增"我的评论"Tab（调用 `/api/users/:uid/comments`）
- 新增"浏览历史"Tab（调用 `/api/users/me/history`）
- 保留原有的"我的收藏"Tab

**Search.tsx：**
- 移除 Firebase Firestore 的 `collection`/`getDocs` 调用
- 搜索改为调用 `/api/search`（支持 q、type、startDate、endDate 参数）
- 热搜词从硬编码改为调用 `/api/search/hot-keywords` 获取
- 新增 `/api/search/suggest?q=` 联想下拉（300ms 防抖）
- 搜索框 onChange 时触发联想建议

### 数据库迁移

**prisma/migrate.sql：**
- `ALTER TABLE Post ADD COLUMN hotScore FLOAT DEFAULT 0`
- `ALTER TABLE Post ADD COLUMN viewCount INT DEFAULT 0`
- `ALTER TABLE WikiPage ADD COLUMN viewCount INT DEFAULT 0`
- `CREATE TABLE Notification (...)`
- `CREATE TABLE BrowsingHistory (...)`
- `CREATE TABLE SearchKeyword (...)`
- 所有迁移均为幂等操作（IF NOT EXISTS / IF NOT EXISTS ALGORITHM=COPY）
