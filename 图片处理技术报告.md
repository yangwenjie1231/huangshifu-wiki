## 2. 图片上传功能实现

### 2.1 上传流程架构

系统采用**会话模式**管理上传流程，确保上传的可靠性和可追踪性：

```
┌─────────┐    创建会话     ┌─────────┐    上传文件     ┌─────────┐
│  客户端  │ ─────────────→ │ 服务端   │ ─────────────→ │ 存储层   │
│         │                │ (Session)│                │         │
└─────────┘                └─────────┘                └─────────┘
     │                           │                           │
     │ ←──────────────────────── │ ←──────────────────────── │
     │      返回会话ID            │      返回文件URL           │
     │                           │                           │
     │ ─────────────────────────→│                           │
     │      完成会话              │                           │
```

### 2.2 核心代码实现

#### 2.2.1 前端上传服务 (imageService.ts)

```typescript
/**
 * 通过会话模式上传图片到 CDN
 * 使用上传会话流程：创建会话 -> 上传文件 -> 完成会话
 */
export const uploadImageToCDNs = async (file: File): Promise<string> => {
  // 1. 计算文件 MD5，用于重复检测
  const md5 = await calculateMD5(file);

  // 2. 检查是否已存在相同 MD5 的图片（去重机制）
  const listResponse = await apiGet<{ items: ImageMap[] }>('/api/image-maps', { md5 });
  const existingItems = listResponse.items || [];

  if (existingItems.length > 0) {
    return existingItems[0].id; // 返回已存在的图片ID
  }

  // 3. 创建上传会话
  const sessionResponse = await apiPost<UploadSessionResponse>('/api/uploads/sessions', {});
  const sessionId = sessionResponse.session.id;

  // 4. 上传文件到会话
  const formData = new FormData();
  formData.append('file', file);

  const uploadUrl = new URL(`/api/uploads/sessions/${sessionId}/files`, window.location.origin);
  const uploadResponse = await fetch(uploadUrl.toString(), {
    method: 'POST',
    credentials: 'include',
    body: formData,
  });

  const uploadData = await uploadResponse.json() as UploadFileResponse;
  const assetId = uploadData.asset.id;

  // 5. 完成会话
  await apiPost(`/api/uploads/sessions/${sessionId}/finalize`);

  return assetId;
};
```

  );

#### 3.1.2 预签名 URL 生成

```typescript
export async function getPresignedUploadUrl(
  key: string,
  expiresIn?: number,
  options?: { contentType?: string; contentMd5?: string; fileSize?: number }
): Promise<{ url: string; key: string; expiresIn: number }> {
  // 1. 验证对象键安全性
  const keyValidation = validateObjectKey(key);
  if (!keyValidation.valid) {
    throw new Error(`对象键验证失败: ${keyValidation.error}`);
  }

  // 2. 验证文件类型和大小
  if (options?.contentType) {
    const typeValidation = validateContentType(options.contentType);
    if (!typeValidation.valid) throw new Error(`文件类型验证失败`);
  }

  // 3. 生成预签名 URL
  const client = getS3ClientWrite();
  const bucket = getPublicBucketConfig();
  const fullKey = bucket.prefix ? `${bucket.prefix}${key}` : key;

  const command = new PutObjectCommand({
    Bucket: bucket.name,
    Key: fullKey,
    ContentType: options?.contentType,
    ContentMD5: options?.contentMd5,
  });

  const url = await getSignedUrl(client, command, { expiresIn: expiry });
  return { url, key: fullKey, expiresIn: expiry };
}
```

### 4.2 Blurhash 解码 Hook (useBlurhash.ts)

```typescript
export function decodeBlurhashToDataURL(
  blurhash: string,
  width = 32,
  height = 32,
  punch = 1
): string | null {
  try {
    // 1. 解码 Blurhash 为像素数据
    const pixels = decode(blurhash, width, height, punch);

    // 2. 创建 Canvas 绘制像素
    const canvas = document.createElement('canvas');
    canvas.width = width;
    canvas.height = height;
    const ctx = canvas.getContext('2d');

    // 3. 将像素数据写入 Canvas
    const imageData = ctx.createImageData(width, height);
    imageData.data.set(pixels);
    ctx.putImageData(imageData, 0, 0);

    // 4. 导出为 DataURL
    return canvas.toDataURL();
  } catch (error) {
    console.error('[Blurhash] Decode error:', error);
    return null;
  }
}
```

## 5. 图片向量嵌入

### 5.1 CLIP 嵌入服务 (clipEmbedding.ts)

系统使用 CLIP (Contrastive Language-Image Pre-training) 模型生成图片的语义向量：

```typescript
import { RawImage, pipeline, CLIPTextModelWithProjection, CLIPTokenizer } from '@xenova/transformers';

const DEFAULT_MODEL_NAME = 'Xenova/clip-vit-base-patch32';
const DEFAULT_VECTOR_SIZE = 512;

// 单例模式缓存模型实例
let imageExtractorPromise: Promise<ExtractorFunc> | null = null;

async function getImageExtractor() {
  if (!imageExtractorPromise) {
    imageExtractorPromise = pipeline('image-feature-extraction', getEmbeddingModelName())
      .then((extractor) => extractor as ExtractorFunc)
      .catch((error) => {
        imageExtractorPromise = null;
        throw error;
      });
  }
  return imageExtractorPromise;
}

export async function generateImageEmbedding(imageBuffer: Buffer): Promise<number[]> {
  const extractor = await getImageExtractor();
  
  // 1. 将图片写入临时文件
  const tmpPath = path.join(os.tmpdir(), `embedding_${Date.now()}.tmp`);
  await fs.promises.writeFile(tmpPath, imageBuffer);
  
  try {
    // 2. 读取图片并提取特征
    const image = await RawImage.read(tmpPath);
    const output = await extractor(image, {
      pooling: 'mean',      // 使用平均池化
      normalize: true,      // 归一化向量
    });

    // 3. 向量归一化
    const vector = normalizeVector(Array.from(output.data!));
    return vector;
  } finally {
    // 4. 清理临时文件
    await fs.promises.unlink(tmpPath).catch(() => {});
  }
}
```

### 5.2 向量同步服务 (embeddingSync.ts)

```typescript
export async function syncImageEmbeddingBatch(
  prisma: PrismaClient,
  uploadsDir: string,
  options: SyncOptions
): Promise<SyncResult> {
  // 1. 获取待处理的图片列表
  const candidates = await prisma.imageEmbedding.findMany({
    where: { status: { in: [EmbeddingStatus.pending, EmbeddingStatus.failed] } },
    include: { galleryImage: { include: { asset: true } } },
    take: options.limit,
  });

  const result: SyncResult = { requested: limit, picked: candidates.length, ready: 0, failed: 0, skipped: 0, details: [] };

  for (const item of candidates) {
    // 2. 解析本地图片路径
    const localPath = resolveLocalImagePath(galleryImage, uploadsDir);
    if (!localPath) {
      result.failed++;
      continue;
    }

    // 3. 读取图片并生成向量
    const imageBuffer = await fs.promises.readFile(localPath);
    const vector = await generateImageEmbedding(imageBuffer);

    // 4. 生成唯一的 Point ID (FNV-1a 哈希)
    const pointId = buildQdrantPointId(item.galleryImageId);

    // 5. 存储到 Qdrant 向量数据库
    await upsertImageEmbeddingPoint({
      pointId,
      vector,
      payload: {
        galleryImageId: item.galleryImageId,
        galleryId: galleryImage.galleryId,
        imageUrl: galleryImage.asset?.publicUrl || galleryImage.url,
        imageName: galleryImage.asset?.fileName || galleryImage.name,
        updatedAt: new Date().toISOString(),
      },
    });

    // 6. 更新数据库状态
    await prisma.imageEmbedding.update({
      where: { id: item.id },
      data: { status: EmbeddingStatus.ready, embeddedAt: new Date() },
    });

    result.ready++;
  }

  return result;
}
```

### 5.3 Point ID 生成算法

使用 FNV-1a 64 位哈希算法生成唯一且确定的 Point ID：

```typescript
const FNV64_OFFSET_BASIS = 0xcbf29ce484222325n;
const FNV64_PRIME = 0x100000001b3n;
const MAX_SAFE_POINT_ID = BigInt(Number.MAX_SAFE_INTEGER);

export function buildQdrantPointId(galleryImageId: string): number {
  const bytes = Buffer.from(galleryImageId, 'utf8');
  let hash = FNV64_OFFSET_BASIS;

  // FNV-1a 哈希计算
  for (const value of bytes) {
    hash ^= BigInt(value);
    hash *= FNV64_PRIME;
    hash &= 0xffffffffffffffffn; // 64位掩码
  }

  // 映射到安全整数范围
  const reduced = hash % MAX_SAFE_POINT_ID;
  const pointId = Number(reduced);
  return pointId > 0 ? pointId : 1;
}
```

// Blurhash 缓存
const blurhashCache = new Map<string, BlurhashResult>();
const BLURHASH_CACHE_TTL = 60 * 60 * 1000; // 1 小时

function getCachedBlurhash(key: string): BlurhashResult | null {
  const cached = blurhashCache.get(key);
  if (cached) {
    const timestamp = blurhashCache.get(`_timestamp_${key}`);
    if (timestamp && Date.now() - (timestamp as any) < BLURHASH_CACHE_TTL) {
      return cached;
    }
    // 过期清理
    blurhashCache.delete(key);
    blurhashCache.delete(`_timestamp_${key}`);
  }
  return null;
}

## 10. 总结

本图片处理系统实现了以下核心能力：

1. **多存储策略**：支持本地、S3、外部图床三重备份，确保数据高可用
2. **智能上传**：MD5 去重、会话管理、进度监控、断点续传
3. **渐进式加载**：Blurhash/Thumbhash 占位图，提升用户体验
4. **语义搜索**：CLIP 向量嵌入，支持以图搜图、语义相似度搜索
5. **安全设计**：路径验证、类型检查、权限分离、预签名 URL

系统采用模块化架构，各组件职责清晰，便于维护和扩展。通过缓存、批量处理、异步化等优化手段，在保证功能完整性的同时实现了良好的性能表现。
